HTML5 crash course

Intro

Deze crash course heeft als doel je snel op weg te helpen met HTML5. Uiteraard is HTML5 enorm uitgebreid en is dit slechts een het tipje van de ijsberg die HTML5 te bieden heeft.

Werkt altijd

HTML5 degrades gracefully, dat wil zeggen dat browsers die HTML niet kennen, het document toch nog kunnen renderen. Dat is de basisfilosofie van HTML5: don't break stuff.

Geen uppercase/lowercase conventies

Vooral bij oudere HTML-documenten zie je sommige elementen in hoofdletters staan:

<SPAN>tekst</SPAN>

In sommige versies van HTML was dit verplicht, in andere mocht het niet. HTML5 is hier duidelijk in: je mag zelf kiezen. Je mag hoofdletters en kleine letters zelfs mengen, al is dat niet meteen een best practice.

Semantiek > layout

Elementen die gebruikt werden om de layout aan te passen en dus geen semantische betekenis hebben, worden afgeraden om te gebruiken. Het gaat hier om <font>, <big>, <strike> en

<center>

Deze kan je makkelijk vervangen door CSS-eigenschappen.

Soms lijken er inconsistenties, zo mag <small> nog wel. Maar, de betekenis van het element is veranderd. <small> wordt gebruikt om aan te duiden dat deze tekst small print is. Bijvoorbeeld de disclaimer bij deze Superman onesie:

<small>Disclaimer: letting your baby wear this does not give him/her ability to fly.</small>

Er is ook wat verwarring rond <i>, <em> en <strong>

Gebruik <em> om de nadruk op iets te leggen.

letting your baby wear this does not give him/her ability to fly.

letting your baby wear this does <em>not</em> give him/her ability to fly.

Gebruik <i> om de toon van een woord te veranderen. Bijvoorbeeld wanneer je iets ironisch bedoeld:

HTML5 is waanzinnig "<i>moeilijk</i>"

Gebruik <strong> om iets als extra belangrijk aan te duiden.

Voor het examen moeten jullie geen HTML5 kennen, maar wel de werking van een deeltjesversneller.

Voor het examen moeten jullie geen HTML5 kennen, maar wel <strong>de werking van een deeltjesversneller</strong>.

Moraal van het verhaal: gebruik HTML5 elementen énkel als er een semantische betekenis bij komt kijken, om de layout te veranderen gebruik je CSS.

KISS

Keep It Simple, Stupid. Er zijn een heleboel vereenvoudigingen in HTML5

Doctype

Moet bovenaan elke HTML-pagina en vertelt de browser om welk type HTML het gaat.Voordien was dit een stevige brok code die niemand van buiten kende. Hier zien we een doctype voor HTML 4.01:

<!DOCTYPE HTML PUBLIC "-//W3C//DTD HTML 4.01//EN" "http://www.w3.org/TR/html4/strict.dtd">

Met HTML5 is dit een pak eenvoudiger:

<!DOCTYPE html>

Character encoding

Ook wel charset genaamd. Vertelt de browser hoe de bits van een bestand naar welke karakters moeten omgezet worden. Er zijn verschillende charsets

  • ASCII (7-bit) : 128 karakters
  • ANSI (8-bit) : 256 karakters
  • UTF8 (min: 1byte, max 4 byte) : maximum 4 294 967 296 karakters
  • ...

Dus hoe meer bits, hoe meer karakters, hoe meer talen er ondersteund kunnen worden. UTF-8 is ondertussen een van de meest gebruikte internationale charsets.

Pre-HTML was dit de notatie om een charset te bepalen:

<meta http-equiv="Content-Type" content="text/html; charset=UTF-8">

En hier is de vereenvoudigde HTML5-versie:

<meta charset="UTF-8">

Script

Ooit al front-end scripting gezien in een andere taal dan JavaScript?

<script type="text/javascript" src="file.js"></script>

Voor browsers bestaat er geen ander scripting-taal dan JavaScript, dus maakt HTML5 het wat simpeler:

<script src="file.js"></script>

Link

Om naar stylesheet te linken moest je voordien vermelden om welk type het ging, hoewel dit al in de relatie vermeld werd: 

<link rel="stylesheet" type="text/css" href="file.css">

HTML5 maakt komaf met deze overkill:

<link rel="stylesheet" href="file.css">

Simpele singletons

Een singleton element is een element dat geen closing-tag bevat, zoals <input>, <hr>en het vaak misbruikte <br>

Vroeg moesten deze elementen met een trailing slash (bv. <hr />) worden afgesloten. In HTML5 zijn deze optioneel en is het zelfs eenvoudiger om ze niet meer te schrijven.

Formulieren

Er zijn een heleboel nieuwe attributen bijgekomen die de verwerking en validatie van formulieren vereenvoudigt. Zo doen browsers nu wat vroeger enkel via JavaScript kon.

Attributen die geen waarde hebben, zoals bv. selected, autofocus, ... moeten dus ook geen waarde toegekend worden, zoals soms wel eens gebeurde voor HTML5: selected="selected".

Placeholder

Wanneer een inputveld geen waarde heeft, kan je door middel van het placeholder=""-attribuut er een waarde in plaatsen die verdwijnt wanneer je het inputveld selecteert, bv:

<input type="text" placeholder="bv. Olaf">

Autofocus

Wanneer een pagina geladen is, kan je automatisch de focus op een bepaald input-veld plaatsen doormiddel van het autofocus-attribuut. Zo kan je onmiddellijk beginnen typen. Denk aan de Google searchbar. Let op: je kan uiteraard maar één autofocus per pagina instellen en van zodra de bezoeker ergens anders klikt verdwijnt de focus.

<input type="text" autofocus>

Noot: deze autofocus staat op off omdat anders bij het laden van het document je telkens op dit veld zou terecht komen in plaats van bij het begin.

Required

Wanneer dit attribuut op een inputveld wordt toegepast, zal de browser valideren of het ingevuld is. Als het leeg is, toont de browser een foutmelding. Probeer het onderstaande formulier te submitten zonder iets in te vullen:

<input type="text" required>

Autocomplete

Browsers kunnen data die de gebruiker heeft ingevuld en doorgestuurd onthouden om deze daarna automatisch weer in te vullen wanneer hij een tijdje later dezelfde pagina bezoekt. Vaak is dit handig, maar soms kan dit ook gevaarlijk zijn, bv. bij gevoelige informatie zoals paswoorden en bankgegevens.

Standaard zet HTML5 een formulier op autocomplete="on", dus als je dit wil vermijden moet je dit expliciet vermelden met autocomplete="off". Dit kan op twee elementen: het formulier, dan zal geen enkel inputveld dat in het formulier zit automatisch ingevuld worden

<form action="/send" autocomplete="off">
	...
</form>

Of op een individueel inputveld:

<input type="text" autocomplete="off">

Input types

HTML5 biedt een heleboel nieuwe input-types met elk hun eigen layout en functie waarbij vooral de de usability voor mobiele gebruikers in acht wordt genomen. Dat wil zeggen dat de layout ook afhankelijk is van browser tot browser.

type="search"

Voor zoekvelden. Toont een kruisje rechts van het zoekveld waarmee je het veld in één klik kan leegmaken.

<input type="search">

type="email"

Controleert of het input-veld een geldig e-mailadres bevat wanneer de gebruiker het formulier verstuurt.

<input type="email">

Voor mobiele gebruikers zal dit ook een toetsenbord oproepen waar automatisch de @ key beschikbaar is.

type="url"

Controleert of het input-veld een geldige url bevat wanneer de gebruiker het formulier verstuurt.

<input type="url">

Voor mobiele gebruikers zal dit ook een toetsenbord oproepen waar automatisch de ., / en .com, keys beschikbaar zijn.

type="tel"

Dit valideert niet of het input-veld een geldig telefoonnummer is, maar roept het numerieke toetsenbord op voor mobiele gebruikers

<input type="tel">

Mobiele gebruikers krijgen een soortgelijk toetsenbord te zien

type="range"

Toont een slider waarvan standaard het minimum 1 en het maximum 100 is en de slider automatisch in het midden van deze waardes wordt gezet.

<input type="range">

Met een beginwaarde van 20:

<input type="range" value="20">

Met andere minimum- en maximumwaarden:

<input type="range" min="2000" max="4000">

type="number"

Aan een type="number" wordt automatisch een spinner toegevoegd, een iconset waarmee je een getal kan optellen of aftrekken.

<input type="number">

Ook op dit element kan een range ingesteld worden. Vergeet de value niet op te geven, anders zal de beginwaarde van de input leeg zijn.

<input type="number" value="10" min="10" max="50">

type="date"

Een type="date" krijgt automatisch een date-picker toegewezen en wordt in het formaat dd/mm/jjjj geschreven

<input type="date">

Er zijn verschillende andere type="date"s, maar let op: afhankelijk van de tijdszone waarin de browser is ingesteld kan de volgorde, het formaat en de taal van de datum-aanduiding verschillend zijn.

type="datetime-local"

Formaat: dd/mm/jjjj uu:mm

Opgelet: type="datetime" zou niet meer ondersteund worden, enkel nog type="datetime-local".

<input type="datetime-local">

type="time"

Formaat: uu:mm

<input type="time">

type="month"

Formaat: maand jjjj

<input type="month">

type="week"

Formaat: Week ww, jjjj

<input type="week">

type="pattern" (Regular Expressions)

Voor wie nog meer flexibiliteit wil is er het pattern-type, waarmee je zelf je patroon kan samenstellen om een bepaald inputveld te valideren. Hiervoor moet je wel regular expressions kennen, maar vaak vind je op het internet copy-pastebare oplossingen voor het merendeel van de datatypes

<input pattern="[\d]{4}">

[\d]{4} is een regular expression die een match oplevert voor 4 achtereenvolgende getallen.

type="color"

Een color-type toont een colorpicker en zal een hexadecimale kleurwaarde teruggeven. Standaard staat de kleur op de waarde #000000, puur zwart.

<input type="color">

nieuwe elementen

De focus van de nieuwe elementen in HTML5 ligt vooral op de semantiek, dat wil zeggen dat deze elementen iets meer vertellen over de functie of hiërarchie van een woord of een tekst in het geheel.

<mark>

Dit element vestigt de aandacht op iets wat belangrijk is voor iemand die iets zoekt, bv. om een zoektermen in een stukje tekst aan te duiden.

<time>

Dit element zorgt ervoor dat een tijdsaanduiding makkelijk geïnterpreteerd kan worden door zowel mensen als computers. In de element value,dat wat tussen de opening en closing tag staat, komt de datum voor mensen, in het datetime-attribuut komt de datum volgens de HTML5 conventie

5 februari 2020 
<time datetime="2020-02-05">5 februari 2020</time>

Opgelet, datetime is onafhankelijk van de locatie en moet dus in het formaat JJJJ-MM-DD geschreven worden.

<meter>

Dit element heeft per se iets met afstand te maken, maar wel met de hoeveelheid van iets. Hier zijn minstens het value en het max attribuut nodig. In de element value kan eventueel een beschrijvende tekst komen.

3.5 TB van 20TB beschikbaar 
<meter value="3.5" max="20">3.5 TB van 20TB beschikbaar</meter>

Deze tekst wordt enkel getoond wanneer de browser geen HTML5 herkent en is een mooi voorbeeld van graceful degradation. je kan dit element nog eindeloos tweaken en er zijn ook interessante layouts in samenwerking met CSS ::before en ::after elementen. Bekijk de mogelijkheden op CSS-tricks: meter element.

<progress>

Dit element lijkt op meter, maar duidt aan hoever bepaalde vorderingen staan.

loading: 75% klaar 
loading: <progress value="75" max="100">75% klaar</progress>

<section>

Dit element zal de functie van een <div> steeds vaker overnemen. Het bundelt stukjes tekst die samenhoren. Het verschil met een <div> is dat een <section> laat weten dat alles wat zich binnen dit element bevindt, samenhoort.

<nav>

Het <nav> element spreekt voor zich: hierin komen alle elementen die instaan voor de navigatie.

  • Over
  • Blog
  • Contact
<nav>
	<ul>
		<li><a href="#">Over</a></li>
		<li><a href="#">Blog</a></li>
		<li><a href="#">Contact</a></li>
	</ul>
</nav>

Dit element gaat vaak gepaard met het <header> element

<header>

Het <header> element wordt omschreven als een verzameling waarvan de elementen dienen tot inleiding of navigatie. Niet te verwarren met het <head> element, dat boven elke body komt en waarin zogenaamde metadata zoals title, charset, css, ... komt te staan.

<hgroup>

Wordt gebruikt wanneer je een groep van titels hebt die in feite samenhoren, zoals een titel en een subtitel. Dit zorgt ervoor dat alle titels dezelfde inspringing krijgen.

Peter Pan

or, The Boy Who Wouldn’t Grow Up

<hgroup>
	<h1>Peter Pan</h1>
	<h2>or, The Boy Who Wouldn’t Grow Up</h2>
</hgroup>

<footer>

Het <footer> element wordt niet gebruikt om informatie aan te duiden die onderaan een sectie komt. Een footer heeft een veel specifiekere functie: het bevat alle informatie over die bepaalde sectie, zoals de auteur, copyright, links naar gerelateerde content, ... Het vertelt iets meer over hoe de bovenstaande tekst tot stand is gekomen, en niets over de inhoud van de tekst zelf.

<aside>

Bij het <aside> element is het niet de positie die van belang is. Een navigatie-item aan de rechter- of linkerkant van een pagina komt niet in aanmerking. Bij HTML5 draait het zoals gewoonlijk om de semantiek, de inhoud, en niet de positie waar iets zich bevindt.

<aside> wordt aanschouwd als een element om iets extra te vertellen over de inhoud van een bepaald deel.

<article>

Het <article> element is de lastigste van de elementen, omdat het erg veel overeenstemming heeft met een <section> element. Het wordt meestal gebruikt voor blogposts, nieuwsartikels, comments, forumposts, reviews waardoor er nog maar weinig over is voor <section> om inhoudelijke elementen te bundelen. En, je mag een <article> ook gebruiken voor zogenaamde widgets, bv. eeneen beursgrafiekje, een twitterfeed.

Ondersteuning

HTML5 wordt niet door oudere browsers ondersteund. HTML5 volledig ondersteunen in deze browsers is moeilijk, maar wat we wel kunnen doen is ervoor zorgen dat de website functioneert, al dan niet met de HTML5 functionaliteit, zolang de gebruiker ongeacht welke browsertechnologie hij gebruikt, maar ongehinderd kan surfen.

CSS

Sommige browsers zullen elementen die ze niet herkennen geen standaard opmaak geven. Daarom is het belangrijk wanneer je met HTML5 werkt, de volgende CSS-code toe te voegen:

section, article, header, footer, nav, aside, hgroup {
	display: block;
}

Wanneer je een reset.css of normalize.css gebruikt zit deze property-definitie er al standaard in verwerkt.

JavaScript

Oude versies van Internet Explorer zullen geen elementen herkennen die niet voorgedefinieerd zijn. Dit wil zeggen dat je voordat de <body> geladen wordt, je alle gebruikte HTML5 elementen eerst via JavaScript moet definiëren.

document.createElement('section');
